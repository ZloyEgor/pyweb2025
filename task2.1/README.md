# Статический сайт на MkDocs с автоматическим деплоем на GitHub Pages

Этот проект демонстрирует создание статического сайта с использованием MkDocs и настройку автоматического деплоя на GitHub Pages через GitHub Actions.

## Результат

**Готовый сайт:** https://zloyegor.github.io/pyweb2025/

## Задание

Развернуть на GitHub Pages статический сайт используя один из движков-генераторов статических сайтов на Python (MkDocs). Автоматизировать процесс деплоя.

### 7. Создание GitHub репозитория

1. Создайте новый репозиторий на GitHub
2. Сделайте его публичным
3. НЕ добавляйте README, .gitignore или лицензию

```bash
# Подключение к GitHub (замените на ваши данные)
git remote add origin https://github.com/zloyegor/pyweb2025.git
git branch -M main
git push -u origin main
```

### 8. Настройка GitHub Actions

Создание файла `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout
      uses: actions/checkout@v4
      
    - name: Setup Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.x'
        
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install "mkdocs>=1.5.0,<2.0.0" "mkdocs-material>=9.0.0"
        
    - name: Build site
      run: |
        cd task2.1/my-static-site
        mkdocs build --verbose --clean
      
    - name: Setup Pages
      uses: actions/configure-pages@v4
      
    - name: Upload artifact
      uses: actions/upload-pages-artifact@v3
      with:
        path: './task2.1/my-static-site/site'

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
    - name: Deploy to GitHub Pages
      id: deployment
      uses: actions/deploy-pages@v4
```

### 9. Настройка GitHub Pages

1. Перейдите в **Settings** → **Pages** вашего репозитория
2. В разделе **Source** выберите **GitHub Actions**
3. Сохраните настройки

### 10. Финальный деплой

```bash
git add .
git commit -m "Add GitHub Actions workflow for automated deployment"
```

## Анализ альтернативных технологических решений

### Применение отечественных CDN для оптимизации доставки контента

Сети доставки контента (Content Delivery Network) представляют собой географически распределенную инфраструктуру серверов, предназначенную для кеширования и доставки статического контента конечным пользователям с минимальными задержками. В контексте российского информационного пространства особое значение приобретают отечественные CDN-провайдеры, обеспечивающие соответствие требованиям информационной безопасности и локализации данных.

CDN77 представляет собой российскую сеть доставки контента с собственной инфраструктурой серверов, развернутой в ключевых точках России и стран СНГ. Платформа поддерживает современные протоколы HTTP/2 и алгоритмы сжатия Brotli, что обеспечивает оптимальную производительность при передаче данных. Интеграция осуществляется посредством REST API, а ценовая модель начинается от $0.049 за гигабайт переданных данных.

Selectel CDN функционирует как российский провайдер с распределенной сетью серверов в более чем 15 городах Российской Федерации. Платформа включает встроенные механизмы защиты от DDoS-атак и поддерживает автоматическую оптимизацию изображений в формате WebP. Тарификация построена по гибкой модели, адаптированной под различные объемы трафика.

Mail.ru Cloud CDN интегрирован в экосистему Mail.ru Group и обеспечивает покрытие территории России, Казахстана и Беларуси. Платформа поддерживает протокол HTTP/3 и предоставляет бесплатный тариф с лимитом до 100 ГБ в месяц, что делает ее привлекательной для небольших проектов.

Yandex Cloud CDN представляет собой часть облачной инфраструктуры Яндекса с интеграцией в Yandex Object Storage. Сеть серверов охватывает Россию и Европу, включает автоматическое сжатие контента и использует модель оплаты по факту потребления ресурсов.

Интеграция CDN в проекты на базе MkDocs осуществляется через конфигурационный файл mkdocs.yml, где указываются внешние ресурсы для CSS и JavaScript файлов, размещенных на CDN-серверах.

### Возможности платформы Gitverse в контексте CI/CD процессов

Gitverse представляет собой отечественную платформу управления исходным кодом и DevOps-процессами, разработанную компанией VK. Платформа позиционируется как альтернатива зарубежным решениям в области управления жизненным циклом разработки программного обеспечения.

Функциональность Git-хостинга включает поддержку неограниченного количества приватных репозиториев, что критично для коммерческих проектов. Платформа поддерживает Git LFS (Large File Storage) для эффективной работы с большими файлами и обеспечивает интеграцию с российскими системами аутентификации и авторизации.

Система непрерывной интеграции и развертывания (CI/CD) в Gitverse построена на основе pipeline-архитектуры с поддержкой Docker-контейнеров. Это обеспечивает изоляцию процессов сборки и возможность параллельного выполнения задач. Интеграция с Kubernetes позволяет масштабировать процессы развертывания в соответствии с требованиями нагрузки.

Конфигурация pipeline для проектов MkDocs осуществляется через файл .gitverse-ci.yml, который определяет этапы сборки и развертывания. Система поддерживает артефакты сборки с настраиваемым временем жизни, что оптимизирует использование дисковых ресурсов.

Дополнительные возможности платформы включают автоматизированное тестирование кода, сканирование на предмет уязвимостей безопасности, интеграцию с российскими облачными провайдерами и соответствие отечественным стандартам информационной безопасности.

### Методологии развертывания статических веб-приложений в производственной среде

Развертывание статических веб-приложений в производственной среде требует комплексного подхода к выбору технологической платформы, учитывающего факторы масштабируемости, надежности, стоимости владения и уровня контроля над инфраструктурой.

Традиционный веб-хостинг представляет собой классический подход к размещению статического контента на выделенных или виртуальных серверах. Данная методология предполагает использование протоколов FTP/SFTP для передачи файлов, утилиты rsync для синхронизации контента и SSH для удаленного администрирования. Российские провайдеры, такие как Reg.ru, Timeweb, Beget и Hostland, предоставляют соответствующие услуги с различными уровнями сервиса. Автоматизация развертывания достигается посредством shell-скриптов, выполняющих последовательность команд сборки и синхронизации.

Облачные платформы представляют более современный подход к хостингу статического контента. Yandex Cloud предоставляет возможность создания S3-совместимых хранилищ с функциональностью статического веб-хостинга через интерфейс командной строки. VK Cloud, базирующийся на технологии OpenStack, обеспечивает создание контейнеров объектного хранилища с поддержкой веб-индексации. Selectel Cloud использует S3-совместимый API, что обеспечивает совместимость с существующими инструментами развертывания.

Контейнеризация на базе Docker представляет собой методологию упаковки приложения и его зависимостей в изолированные контейнеры. Для статических сайтов типичным решением является использование образа nginx:alpine в качестве базового, с последующим копированием статических файлов в соответствующий каталог. Docker Compose обеспечивает декларативное описание многоконтейнерных приложений, а Kubernetes предоставляет возможности оркестрации контейнеров в кластерной среде с автоматическим масштабированием и балансировкой нагрузки.

Serverless архитектура представляет парадигму вычислений, при которой облачный провайдер динамически управляет распределением машинных ресурсов. Yandex Cloud Functions и VK Cloud Functions предоставляют среды выполнения для JavaScript и Python, позволяя обслуживать статический контент через функции, активируемые HTTP-запросами. Данный подход обеспечивает автоматическое масштабирование и оплату только за фактически потребленные ресурсы.

JAMstack (JavaScript, APIs, Markup) представляет современную архитектуру веб-разработки, ориентированную на предварительную генерацию статического контента. Платформы Vercel и Netlify предоставляют интегрированные решения для сборки и развертывания JAMstack-приложений с автоматической интеграцией систем контроля версий и глобальными CDN.

Собственная инфраструктура предполагает полный контроль над всеми компонентами системы хостинга. Технический стек включает веб-серверы (Nginx, Apache HTTP Server), балансировщики нагрузки (HAProxy), системы мониторинга (Prometheus, Grafana) и централизованного логирования (ELK Stack). Конфигурация Nginx включает оптимизации кеширования, сжатия контента и заголовков безопасности.

Сравнительный анализ решений демонстрирует различные компромиссы между сложностью реализации, стоимостью владения, возможностями масштабирования и уровнем контроля. Традиционный хостинг характеризуется низкой сложностью и стоимостью, но ограниченными возможностями масштабирования. Облачные платформы обеспечивают высокую масштабируемость при средней сложности и стоимости. Контейнеризация требует высокой экспертизы, но предоставляет максимальную гибкость. Serverless решения минимизируют операционные затраты, но ограничивают контроль над инфраструктурой.
